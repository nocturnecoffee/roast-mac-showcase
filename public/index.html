Roast-Mac

Reverse-engineered controller · v1

反編譯商用熱風機原廠軟體,
自建 macOS 無線遠端控制器。

原廠 ROAST-Master 控制軟體只有 Windows .exe,必須插著機器才能用,電腦斷線烘焙也斷。 我用 Claude Code 反編譯原廠 .NET 組件、重建 Modbus TCP 協議,用 400 行 Python 從零寫跨平台替代品 —— 把筆電收起來,Mac 在任何房間透過 WiFi 橋接器連 192.168.11.20 就能遙控烘豆,斷線機器繼續照配方跑,只是螢幕看不到而已。

.NET 反編譯 Modbus TCP Python · pymodbus Cross-platform WiFi Bridge Built with Claude Code
UI 操作實錄 — 即時 BT/ET 曲線、15 段配方引擎、Stage Markers 爆點記錄。整套可跑在 macOS / Linux / Windows 任何有 Python 的環境。

起點

原廠軟體的三個痛點

原廠 ROAST-Master.exe
  • Windows only · 沒有 macOS / Linux 版
  • USB 直連,電腦必須跟機器同桌
  • UI 是舊式 WinForms,無遠端存取
  • 斷電/斷線沒有自動恢復機制
  • 配方儲存是扁平檔,難組織
Roast-Mac 替代品
  • Python + web UI,跨全平台
  • TP-Link WiFi 橋接,同房網段就能控制
  • 任何裝置開瀏覽器:Mac / iPad / iPhone
  • 機器記憶配方,斷網繼續烘,只是看不到
  • 階層式配方管理 + 版本/別名

流程

Claude Code 如何完成反編譯 → 替代軟體

  1. 解包原廠 .NET 組件 用 ilspy / dnSpy 反編譯 Roast-Master.exe + 相關 DLL,Claude Code 分析 C# source 找出 Modbus register 映射與功能 code。
  2. 建立 register 字典 從反編譯結果推斷 reg 0-59 = 15 段配方 × [TA/TB/Fan/Time],reg 75-77 = 預熱 setpoint,reg 100-299 = PLC 狀態/感測器,reg 104=1 啟動、reg 148=1 停止。
  3. 確認 wire protocol 用 Wireshark 抓原廠軟體 ↔ 機器的封包,跟反編譯結果互驗,鎖定 Modbus TCP / port 502 / device_id=1。
  4. 寫自動 register 偵測 即時感測位置因 PLC 韌體版本不同,寫了「三連 register [temp, temp, fan] 模式 + ×10 編碼偵測 + scoring」的 auto-discovery,連線就自動鎖定 BT/ET/Fan register。
  5. 400 行 Python pymodbus 3.13 + stdlib http.server,無框架依賴。前端單頁 HTML + Canvas 即時曲線,500 ms polling。
  6. 校準 BT 探針老化 熱電偶隨使用時數讀值逐漸偏低。UI 加動態 bt_offset,補正顯示值;15 段配方目標同時標記「真實 BT / 讀值 BT」雙軌。
  7. WiFi 橋接組網 機器 Ethernet 固定 IP 192.168.11.20 → TP-Link 迷你 AP 橋接 → Mac 連該 SSID,跟機器同網段就能 Modbus TCP。

技術細節

反編譯得到的協議規格(給同機型用戶參考)

通訊協議
Modbus TCP,port 502,device_id=1 (pymodbus 3.13 的 device_id 參數,對應舊版 slave)
Register 分區
0-59 配方 (15 段 × 4 欄)· 60-74 延伸參數 · 75-77 預熱 setpoint · 80-93 ASCII 機器名 · 100-199 PLC 狀態 · 200-299 感測器與歷史緩衝
配方寫入
write_registers(address=0, values=[TA,TB,Fan,Time]*15, device_id=1),每段 4 值共 60 registers
即時感測
三連 register [ET, BT, Fan],溫度 ×10 編碼(register 值 1923 = 192.3°C)
Auto-detect
掃描 94-298 找連續 [temp, temp, fan] triplet,scoring 加分:×10 編碼 +40、ET>BT +15、合理溫差 +20、MAX 設定 -30
啟動 / 停止
寫 reg 104 = 1 啟動烘焙指令;寫 reg 148 = 1 停止。一旦啟動,機器靠 PLC 內建配方緩衝跑,WiFi 斷線不影響烘焙繼續
歷史緩衝
reg 221-238 存 9 個 ET/BT 時序快照,機器自動寫入;重連後可取回補齊 UI 曲線
豆量縮放
配方 Fan 欄上傳前依豆量自動縮放:600g=100%, 450g=85%, 300g=75%, 150g=62% — 規則來自反編譯的原廠對照表

給同好

同機型使用者可直接取用

本機安裝只需要 Python 3.10+ 與 pymodbus 3.13。 Fork 以下 repo,在同網段的電腦上跑 python3 roast_mac.py,打開 http://localhost:8081 就能控制烘豆機。 WiFi 橋接器任何 TP-Link 迷你 AP 都可以(TL-WR802N 最便宜)。

Modbus register 映射表跟反編譯得到的協議規格列在技術細節章節,給想自己寫 iOS / Android App 或整合到 Home Assistant / OpenHAB 的人直接取用。
